# Capacitor iOS App Implementation Plan

## Overview
Wrap the Untangle web app with Capacitor to ship on the iOS App Store. The app calls the existing Railway backend — no Flask server bundled locally.

**Architecture decision: bundled assets** (`dist/` copy of Flask template + static files). API calls go to Railway via absolute URLs. This avoids the 4.2 "thin wrapper" rejection risk of remote URL loading.

## Phase 1 — Capacitor Shell + Backend Wiring
*Goal: web app loads in native iOS shell, all API calls reach Railway.*

### New files to create
- `ios-app/package.json`
- `ios-app/capacitor.config.ts`
- `ios-app/build.sh` — copies Flask template + static into `dist/`, rewrites asset paths
- `ios-app/dist/index.html` — built HTML with `API_BASE` injection
- `ios-app/dist/static/` — copied static assets
- `ios-app/ios/` — generated by `npx cap add ios` (do not hand-edit)

### Key changes
**`app.js`** — add near top:
```js
const IS_NATIVE = !!(window.Capacitor);
const API_BASE = IS_NATIVE
  ? 'https://time-estimation-agent-production.up.railway.app'
  : '';
```
Then rewrite all 9 `fetch('/api/...')` calls to `fetch(API_BASE + '/api/...')`.

**`web_app.py`** — add flask-cors:
```python
from flask_cors import CORS
CORS(app, origins=['capacitor://localhost', 'ionic://localhost', 'http://localhost'])
```

**`requirements.txt`** — add `flask-cors`

**`style.css`** — safe area insets (Dynamic Island / notch):
```css
.app-header {
  padding-top: calc(14px + env(safe-area-inset-top));
}
body {
  padding-bottom: env(safe-area-inset-bottom);
}
```

**`index.html`** — update viewport meta:
```html
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover" />
```

**`capacitor.config.ts`**:
```ts
const config: CapacitorConfig = {
  appId: 'com.untangle.app',
  appName: 'Untangle',
  webDir: 'dist',
  server: {
    allowNavigation: ['time-estimation-agent-production.up.railway.app'],
  },
  ios: {
    contentInset: 'automatic',
    scrollEnabled: false,
  },
};
```

### Milestone
App loads in Xcode simulator → welcome screen appears → brain dump → plan renders from Railway.

## Phase 2 — Native Microphone ⚠️ Critical for App Store
*Replace Web Audio API with Capacitor Voice Recorder. Apple must see a real native AVAudioSession permission dialog.*

**Plugin:** `@capacitor-community/voice-recorder`

**`Info.plist`** — add:
```
NSMicrophoneUsageDescription → "Untangle needs the microphone to let you voice-input your plans."
```

**`app.js`** — rewrite `startRecording`/`stopRecording` with native branch:
- `IS_NATIVE`: use `VoiceRecorder.startRecording()` / `stopRecording()`
- Fallback: existing Web Audio API path (unchanged, for browser)
- Native returns base64 AAC blob → convert to `FormData` → POST to existing `/api/transcribe`

**`web_app.py`** — update `api_transcribe` to handle `audio/aac` / `audio/mp4` mime types.

### Milestone
Tap mic on physical iPhone → native iOS permission sheet appears → voice recorded → transcription appears.

## Phase 3 — Local Notifications
*Schedule "your next block starts in 5 min" notifications on plan approval. Strongest defense against 4.2 thin-wrapper rejection.*

**Plugin:** `@capacitor/local-notifications`

**`Info.plist`** — add:
```
NSUserNotificationUsageDescription → "Untangle sends reminders when your next time block is about to start."
```

**`app.js`** — add `scheduleBlockReminders(timeBlocks)` function:
- Called inside `approvePlan()` after approval
- Schedules one notification per block, 5 min before start time
- Also cancel notifications on "Start fresh"

### Milestone
Approve a plan → notification permission prompt → notification arrives 5 min before a block.

## Phase 4 — ICS / Calendar Export Fix
*Server-round-trip ICS export doesn't trigger native Calendar dialog inside WKWebView.*

**Plugins:** `@capacitor/filesystem`, `@capacitor/share`

**`app.js`** — rewrite `confirmAppleCalExport` for native:
```js
if (IS_NATIVE) {
  const result = await Filesystem.writeFile({
    path: fileName, data: ics, directory: Directory.Cache, encoding: 'utf8',
  });
  await Share.share({ title: 'Untangle Schedule', url: result.uri, dialogTitle: 'Add to Calendar' });
  return;
}
// existing browser fallbacks unchanged
```

### Milestone
Tap "Add to Apple Calendar" on iPhone → native share sheet → Calendar imports all events.

## Phase 5 — Google Calendar OAuth
*OAuth redirect breaks inside WKWebView. Skip in v1; re-add in v1.1.*

**v1 plan:** Disable Google Calendar button in native context. Show "Open untangle.app in Safari to connect Google Calendar."

**v1.1 plan:** Register custom URL scheme (`untangle://gcal-callback`), use `@capacitor/browser` for OAuth, handle `appUrlOpen` event.

## Phase 6 — App Store Polish
*Pass review. Primary risks: 4.2 thin-wrapper, permission string rejections, crash on reviewer device.*

- [ ] **Splash screen**: `@capacitor/splash-screen`, `backgroundColor: '#f4eee6'`, no spinner, 800ms
- [ ] **Offline message**: if `!navigator.onLine` on startup, show "Untangle needs internet to plan your day" instead of blank screen
- [ ] **Guard GA4 `history.replaceState`**: in Capacitor context `window.location` is `capacitor://localhost` — URL manipulation causes WKWebView errors. Add `if (IS_NATIVE) return;` at top of `updateVirtualLocation()`
- [ ] **Hide keyboard hint**: `.kbd-hint { display: none; }` in native context
- [ ] **App Store metadata**:
  - Category: Productivity
  - Subtitle: AI-powered day planning
  - Description: emphasize voice input, block reminders, calendar export (native capabilities)
  - Screenshots: iPhone 15 Pro (6.7") + iPhone SE (4.7")
  - Age rating: 4+
  - Privacy policy URL: `https://time-estimation-agent-production.up.railway.app/privacy`

### App Store rejection risk matrix
| Risk | Mitigation |
|---|---|
| 4.2 thin wrapper | Native mic + notifications + calendar export |
| 2.5.1 wrong mic permission | Phase 2: VoiceRecorder plugin (AVAudioSession) |
| 5.1.1 missing privacy policy | `/privacy` route on Railway |
| 2.1 crashes | TestFlight external beta first |
| 4.5.4 web browser within app | Disable GCal OAuth in v1 |

## Architectural decisions

| Decision | Choice | Reason |
|---|---|---|
| Bundled vs. remote URL | **Bundled** | Remote URL risks 4.2 rejection |
| Google Calendar in v1 | **Skip** | OAuth flow too complex; add in v1.1 |
| Session persistence | v1 uses Flask cookies (fine for now) | v1.1: migrate to `@capacitor/preferences` |
